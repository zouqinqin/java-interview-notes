# 泛型

> 练习代码：[Step4_TypeErasure.java](../src/reflect_proxy/Step4_TypeErasure.java)

## 一、泛型擦除是什么

Java 泛型只在**编译期**有效，编译后字节码中泛型信息被完全删除，这就是泛型擦除。

```java
List<String> listA = new ArrayList<>();
List<Integer> listB = new ArrayList<>();

listA.getClass() == listB.getClass()  // true，运行时都是 ArrayList
```

---

## 二、类型安全靠谁保证

| 阶段 | 做的事 |
|------|--------|
| 编译期 | 编译器做类型检查，拒绝 `listA.add(123)` |
| 编译期 | 编译器在取值处自动插入 `checkcast` 强转指令 |
| 运行期 | JVM 执行 `checkcast`，类型不匹配抛 `ClassCastException` |

JVM 不知道泛型的存在，安全性完全由编译器保证。

---

## 三、反射可以绕过泛型检查

```java
List<String> list = new ArrayList<>();

// 编译器拦住：list.add(123);

// 反射绕过编译器，直接操作字节码
list.getClass().getMethod("add", Object.class).invoke(list, 123);
// 塞进去成功，但取出来会 ClassCastException
String s = list.get(0);  // 爆炸：Integer cannot be cast to String
```

## 四、面试高频题

**Q1：Java 泛型是真泛型还是伪泛型？**
> 伪泛型。编译后类型信息被擦除，运行时 `List<String>` 和 `List<Integer>` 没有区别，都是 `ArrayList`。C# 是真泛型，运行时保留类型信息。

**Q2：泛型擦除后类型安全靠什么保证？**
> 编译器负责类型检查，并在取值处自动插入强转指令（`checkcast`）。运行时如果类型不匹配会抛 `ClassCastException`。

**Q3：`List<String>` 和 `List<Integer>` 能用 `instanceof` 区分吗？**
> 不能。运行时两者都是 `ArrayList`，`instanceof` 无法区分泛型参数。

**Q4：怎么在运行时获取泛型的类型信息？**
> 通过 `ParameterizedType` 反射获取，适用于类字段或方法返回值上的泛型声明，局部变量的泛型无法获取。

```java
// 例：获取字段的泛型类型
Field field = Foo.class.getDeclaredField("list");
ParameterizedType pt = (ParameterizedType) field.getGenericType();
Type actualType = pt.getActualTypeArguments()[0];  // String
```
