## 一、反射

### 核心步骤

```java
// 1. 获取方法（注意：基本类型用 int.class，不是 Integer.class）
Method method = Calculator.class.getDeclaredMethod("add", int.class, int.class);

// 2. 跳过访问权限检查（private 方法必须加）
method.setAccessible(true);

// 3. 调用
Object result = method.invoke(obj, 3, 5);
```

### 关键点

- `int.class` ≠ `Integer.class`，反射不做自动装箱
- 不加 `setAccessible(true)` 调用 private 方法会抛 `IllegalAccessException`
- Java 9+ 模块系统对跨模块反射有额外限制

## 二、JDK 动态代理

### 原理

运行时动态生成接口的实现类（`$Proxy0`），继承自 `Proxy`，绑定 `InvocationHandler`，所有接口方法调用都被拦截转发到 `invoke()`。

### 代码结构

```java
// 1. 实现 InvocationHandler
class LogHandler implements InvocationHandler {
    private final Object target;
    LogHandler(Object target) { this.target = target; }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("[LOG] " + method.getName() + " 开始");
        Object result = method.invoke(target, args);  // target 是真实对象
        System.out.println("[LOG] " + method.getName() + " 结束");
        return result;
    }
}

// 2. 创建代理
ICalculator proxy = (ICalculator) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    new Class[]{ICalculator.class},
    new LogHandler(target)
);
```

### 限制

- 目标类**必须实现接口**，代理的是接口方法
- `$Proxy0` 已继承 `Proxy`，Java 单继承，无法再继承目标类

## 三、CGLIB 动态代理

### 原理

运行时动态生成目标类的**子类**（`Calculator$$EnhancerByCGLIB$$...`），方法调用被拦截转发到 `MethodInterceptor.intercept()`。

### 代码结构

```java
// 1. 实现 MethodInterceptor
class LogInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("[LOG] " + method.getName() + " 开始");
        Object result = proxy.invokeSuper(obj, args);  // 必须用 invokeSuper，不能用 method.invoke(obj)
        System.out.println("[LOG] " + method.getName() + " 结束");
        return result;
    }
}

// 2. 创建代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(Calculator.class);
enhancer.setCallback(new LogInterceptor());
Calculator proxy = (Calculator) enhancer.create();
```

### 限制

- **final 类/方法**无法代理（无法继承）
- `intercept` 里必须用 `proxy.invokeSuper(obj, args)`，用 `method.invoke(obj)` 会无限递归导致 `StackOverflowError`

---

## 四、对比总结

| | JDK 动态代理 | CGLIB |
|--|--|--|
| 原理 | 生成接口实现类 | 生成目标类子类 |
| 要求 | 必须有接口 | 不需要接口 |
| 限制 | 只能代理接口方法 | final 类/方法不可代理 |
| 递归风险 | 无 | invokeSuper 才安全 |
| Spring 使用时机 | 目标类有接口时 | 目标类无接口时 |

## 五、面试高频题

**Q1：JDK 动态代理为什么需要接口？**
> `$Proxy0` 继承了 `Proxy` 类，Java 单继承限制，只能再实现接口，无法继承目标类。

**Q2：CGLIB 代理 final 类为什么报错？**
> CGLIB 原理是生成子类，final 类禁止被继承，直接抛 `Cannot subclass final class`。

**Q3：CGLIB 的 `invokeSuper` 和 `method.invoke` 有什么区别？**
> `invokeSuper` 调用父类（真实对象）方法，跳过拦截器；`method.invoke(obj)` 中 obj 是代理对象，会再次触发拦截器，造成无限递归。

**Q4：Spring AOP 用的是哪种代理？**
> 目标类实现了接口 → JDK 动态代理；没有接口 → CGLIB。Spring Boot 2.0 后默认优先使用 CGLIB。

**Q5：动态代理和静态代理的区别？**
> 静态代理：手动编写代理类，编译时确定，每个类都要写一个代理类，维护成本高。
> 动态代理：运行时自动生成，一个 Handler 可以代理任意类，Spring AOP 就是基于此实现的。

**Q6：反射性能差在哪？**
> 需要动态解析类型，无法被 JIT 内联优化；每次 `invoke` 都有权限检查开销（`setAccessible(true)` 可减少但不消除）。
